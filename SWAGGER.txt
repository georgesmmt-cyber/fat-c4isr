## 2. **BASE DE DONNÉES SUPABASE**

```sql
-- Table des utilisateurs
users (
    id UUID PRIMARY KEY,
    username TEXT,
    password_hash TEXT,
    role TEXT,
    full_name TEXT,
    email TEXT,
    avatar_url TEXT,
    is_active BOOLEAN,
    created_at TIMESTAMP,
    timestamp TIMESTAMP
)

-- Table des alertes
alerts (
    id SERIAL PRIMARY KEY,
    vest_id TEXT,
    type TEXT,
    severity TEXT,
    message TEXT,
    time TIMESTAMP,
    timestamp TIMESTAMP,
    acknowledged BOOLEAN
)

-- Table des lectures capteurs
sensor_readings (
    id SERIAL PRIMARY KEY,
    vest_id TEXT,
    status TEXT,
    battery_level INTEGER,

    last_seen TIMESTAMP
)
```

## 6. **CE QUI MANQUE / À AMÉLIORER**

### ❌ **MANQUANT CRITIQUE**

| Élément | Raison | Solution |
|---------|--------|----------|
| **CHAT** | Table manquante + pas d'UI | Créer table `chat_messages` + composant HTML |
| **Table `chat_messages`** | Pas dans Supabase | Exécuter le SQL fourni |
| **Interface chat** | Pas dans ton HTML | Ajouter le composant (fourni plus haut) |
| **Routes API chat** | Pas dans backend | Ajouter endpoints `/api/chat/*` |

### ⚠️ **À AMÉLIORER**

| Élément | Problème | Solution |
|---------|----------|----------|
| **Supabase Auth** | Auth custom dans backend | Migrer vers Supabase Auth |
| **Row Level Security** | Pas configuré | Ajouter politiques RLS |
| **Realtime** | Pas activé sur les tables | Activer dans interface Supabase |
| **PostGIS** | Non installé | Installer pour fonctionnalités géospatiales |
| **Firebase** | Initialisé mais inutilisé | Supprimer ou utiliser vraiment |
| **Documentation** | Swagger présent mais à compléter | Documenter tous les endpoints |
| **Tests unitaires** | Aucun | Ajouter tests Jest |
| **CI/CD** | Manuel | Automatiser avec GitHub Actions |

### 🔧 **OPTIMISATIONS POSSIBLES**

| Optimisation | Bénéfice |
|--------------|----------|
| Migrer vers Supabase JS client | Accès aux fonctionnalités Realtime/Auth |
| Compression gzip sur backend | Réduction bande passante |
| Lazy loading des onglets | Chargement plus rapide |
| PWA (Service Worker) | Mode hors-ligne avancé |
| WebSocket secure (WSS) | Sécurité en production |

## 7. **ÉTAT DE LA MIGRATION WEB**

### **Actuellement**
- ✅ Backend local sur `localhost:5001`
- ✅ Frontend accessible via `file://` ou `http://localhost:5001`
- ✅ Base de données sur Supabase (cloud)
- ❌ Pas accessible depuis l'extérieur

### **Ce qui est prêt pour le web**

```javascript
// Dans ton HTML (ligne ~80)
const API_BASE = window.location.protocol === 'file:' 
    ? 'http://localhost:5001/api'   // Mode local
    : '/api';                        // Mode web (même domaine)

// Détection automatique de l'environnement
window.ENV = {
    isLocal: hostname === 'localhost',
    isProduction: !(hostname === 'localhost'),
    apiUrl: isLocal ? 'http://localhost:5001/api' : 'https://api.fat-c4isr.com/api'
};
```

### **URLs à modifier pour le web**

| Fichier | Ligne | Actuel | À remplacer par |
|---------|-------|--------|-----------------|
| HTML | ~80 | `'http://localhost:5001/api'` | `'/api'` (ou domaine absolu) |
| HTML | ~2100 | `'ws://localhost:5001'` | `'wss://api.fat-c4isr.com'` |
| HTML | ~2150 | `'ws://localhost:5001'` | `'wss://api.fat-c4isr.com'` |
